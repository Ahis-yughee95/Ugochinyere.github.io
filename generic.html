Data Pre-Processing and Visualization for
Customer Records Analysis

Project Overview:
This project involved pre-processing a large dataset of customer records to prepare it for further analysis and generate insights for business decision-making. I utilized Python’s core libraries (without using pandas or seaborn for data handling) and structured the data to meet analytical needs, handled errors in the dataset, derived metrics, and saved outputs in JSON format. Visualizations and additional analysis were conducted using pandas and seaborn.

Key Project Phases:

  1. Data Ingestion and Structure Transformation:
    • Read customer records from a CSV file using Python’s CSV library.
    • Transformed flat data into nested structures to represent entities like Vehicle, Credit Card, and Address, enhancing the richness of the data.
  2. Data Cleaning and Error Handling:

    Addressed errors in the dataset, such as empty values in the "dependents" field, replacing them with meaningful defaults and logging the rows where these corrections were made.

  3. Output Generation:
    • Generated structured JSON files for easy data retrieval:
      • processed.json: containing all customer records.
      • retired.json and employed.json: segmented by employment status.
      • remove_ccard.json: flagged customers with credit cards exceeding a 10-year validity span, per client requirements.
  4. Additional Metrics Calculation:
    • Added a "Salary-Commute" metric representing the salary per mile of commute. Special handling was applied for customers traveling one mile or less to reflect salary alone.
    • Sorted records by this new metric in ascending order and saved as commute.json for ranking purposes.
  5. Data Visualization:
      With pandas and seaborn, I generated key visual insights:
      • Calculated the mean salary and median age of customers.
      • Plotted age distributions, dependents, and age conditioned by marital status.
      • Created multivariate plots for salary against commute distance, age against salary, and age against salary conditioned by dependents.
      • Enabled saving of visualizations to provide a reusable asset for client presentations.

Code Structure and Presentation:
The code was modular, with logical structure, clear variable names, and error handling to ensure robustness. Markdown cells were included to delineate each task, making the notebook readable and maintainable for future enhancements.
