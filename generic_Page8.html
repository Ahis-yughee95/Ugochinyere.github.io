Emergency Vehicle Identification

Project Overview:
This report focuses on classifying vehicles into emergency and non-emergency categories by analyzing vehicle characteristics and components. The aim is to enhance road access for emergency vehicles, ensuring better response times during critical situations.

Data Overview and Pre-processing

The dataset, obtained from an online source, includes images and labels indicating whether vehicles are emergency or non-emergency. The data underwent several pre-processing steps to ensure quality and consistency:

  • Data Source: The dataset was sourced from an online platform containing vehicle images and labels.
  • Data Pre-processing: Columns were converted to ‘object’ data types, and image shapes were standardized. Image augmentation was applied using Keras Image Data Generator.
  • Data Split: The data was split into 70% for training and 30% for validation.

Convolutional Neural Network (CNN)

A CNN model was used for classifying the vehicles. Key features of the model include:

  • Convolution Layers: Filters with ReLU activation functions, MaxPooling for feature selection, and dropout to prevent overfitting.
  • Hyperparameters: 32 and 64 filters with kernel sizes of 3x3, padding set to “same” to retain the feature map.
  • Activation Function: Sigmoid function at the output layer for binary classification.

Model Compilation and Training

The CNN model was compiled using the Adam optimizer and binary cross-entropy loss function. It was trained for 10 epochs using the training and validation datasets.

Model Evaluation

The model’s performance was evaluated using accuracy metrics to measure how well the predicted labels matched the vehicle features:

  • Accuracy: The accuracy increased as more convolution layers were added, but some overfitting occurred as the layers deepened.
  • Overfitting: The widening gap between training and validation losses suggests that the model may have overfit with additional layers.

Key Findings

  • Improved Classification: The model demonstrated improved classification accuracy with each convolution layer.
  • Overfitting: A widening gap in loss/accuracy between training and validation data indicates potential overfitting as layers increased.

Conclusion

This report demonstrated the application of a CNN model for classifying vehicles into emergency and non-emergency categories. While the model showed improvement in accuracy, the overfitting issue suggests that further optimization and testing may be required to enhance its performance and ensure better classification for real-time emergency scenarios.
